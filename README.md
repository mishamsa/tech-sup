## 📖 Стандарти документування коду

Для підтримки чистоти коду та автоматичного оновлення документації ми використовуємо стандарт **JSDoc**.

### Як правильно документувати:
1. **Коментарі:** Всі функції, класи та методи повинні мати блок `/** ... */`.
2. **Теги:**
   - `@param {type} name - опис` — обов'язково для всіх вхідних аргументів.
   - `@returns {type} опис` — обов'язково для всіх функцій, що повертають значення.
   - `@async` — якщо функція повертає Promise.
   - `@throws {Error}` — якщо функція може генерувати помилки.
3. **Мова:** Документація ведеться українською мовою для відповідності вимогам бакалаврської роботи.

### Генерація документації:
