# Telegram-бот для автоматизації турнірів з настільних ігор

Бакалаврський проєкт Краснокутського Міхаїла

## Опис проекту

Цей проект призначений для автоматизації процесів організації турнірів через Telegram-бота. Система дозволяє реєструвати учасників, генерувати турнірні сітки за швейцарською системою та системою на вибування, а також вести статистику ігор у режимі реального часу.

---

## Інструкція з розгортання для розробників

Ця інструкція розрахована на встановлення проекту на чисту операційну систему.

### 1. Необхідне програмне забезпечення

Для роботи з проектом необхідно встановити:
* Git: Інструмент для контролю версій. Завантажити з https://git-scm.com/
* Node.js (версія 18.0.0 або вище): Платформа для запуску JavaScript. Завантажити з https://nodejs.org/
* Visual Studio Code: Рекомендований редактор коду. Завантажити з https://code.visualstudio.com/

### 2. Клонування репозиторію та налаштування

Відкрийте термінал (або командний рядок) та виконайте наступні кроки:

1. Клонуйте проект на локальний комп'ютер:
   git clone <посилання_на_репозиторій>

2. Перейдіть у директорію проекту:
   cd tournament-landing

3. Встановіть усі необхідні залежності:
   npm install

### 3. Налаштування середовища розробки

Для дотримання стандартів коду в редакторі VS Code рекомендується встановити розширення:
* ESLint: Для автоматичної перевірки синтаксису JavaScript.
* Stylelint: Для перевірки правильності написання CSS.

### 4. Робота з базою даних

Фронтенд-частина проекту (Landing Page) працює з даними через зовнішні API, тому локальне встановлення SQL-сервера для візуальної частини не потрібне. Для роботи бекенд-частини (бота) використовується SQLite, яка постачається разом із кодом і не потребує додаткового налаштування сервера.

### 5. Запуск проекту та базові команди

* Перевірка JavaScript коду на помилки:
  npm run lint

* Перевірка CSS файлів:
  npm run lint:css

* Автоматичне виправлення помилок стилю:
  npm run lint:fix

* Перегляд проекту:
  Відкрийте файл index.html у браузері або скористайтеся розширенням Live Server у VS Code.

## Стандарти та рекомендації щодо документування коду

У проекті прийнято стандарт JSDoc для опису логіки JavaScript.

### Основні правила:
1. Кожна ключова функція, метод або клас повинні мати опис безпосередньо перед оголошенням.
2. Для опису параметрів використовується тег @param з вказанням типу даних у фігурних дужках.
3. Для опису результату, що повертає функція, використовується тег @returns.
4. Для асинхронних функцій обов'язково додається тег @async.

### Приклад документування:

/**
 * Розраховує рекомендовану кількість раундів турніру.
 * @param {number} participantCount - Кількість учасників турніру.
 * @returns {number} Кількість раундів.
 */
function calcSwissRounds(participantCount) { ... }

### Автоматична генерація документації:

Для створення повноцінної технічної документації у форматі HTML виконайте команду:
npm run docs

Згенеровані файли будуть доступні в папці /docs. Усі нові розробники повинні дотримуватися цього стандарту при додаванні нових функцій.
