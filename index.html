Актуальність теми

Настільні ігри переживають глобальне відродження: кількість учасників організованих змагань щорічно зростає. Клуби, федерації та ігрові магазини щомісяця організовують десятки турнірів — від локальних до міжнародних.

Проте переважна більшість організаторів досі покладається на таблиці Excel, паперові жеребкування та розрізнені месенджер-чати. Це призводить до помилок, затримок і незадоволеності учасників.

Існуючі рішення або надто спеціалізовані (лише шахи/го), або надто дорогі для аматорських спільнот. Розрив між потребою та доступним інструментарієм — і є полем для даного дослідження.

02

Мета та завдання

Мета дослідження

Розробити та реалізувати інформаційну систему, яка автоматизує повний цикл управління турнірами з настільних ігор: від реєстрації учасників до формування підсумкових рейтингів, забезпечуючи масштабованість, зручність та надійність.

  1. 1

    Аналіз предметної галузі

    Дослідити існуючі системи управління турнірами, виявити їх переваги та недоліки, сформулювати вимоги до нового рішення.

  2. 2

    Проектування архітектури

    Розробити структуру бази даних та архітектуру серверної частини з урахуванням вимог до продуктивності та масштабованості.

  3. 3

    Реалізація модулів жеребкування

    Імплементувати алгоритми Swiss, Round-Robin, Single/Double Elimination для різних форматів змагань.

  4. 4

    Розробка інтерфейсу користувача

    Створити адаптивний веб-інтерфейс для організаторів, учасників та глядачів із реальним часом оновлень.

  5. 5

    Тестування та оцінка ефективності

    Провести функціональне та навантажувальне тестування, порівняти з існуючими аналогами.

03

Методологія дослідження

🔍

Аналітичні методи

Порівняльний аналіз існуючих систем (Challonge, Toornament, Swiss-Manager), метод структурного аналізу вимог, UML-моделювання.

⚙️

Технічна реалізація

Agile-методологія розробки (Scrum-спринти), розробка через тестування (TDD), REST API як основний протокол взаємодії компонентів.

🧮

Алгоритмічні методи

Дослідження та реалізація алгоритмів жеребкування: Monrad Swiss, Dutch Swiss, Round Robin. Алгоритми оцінки рейтингу Elo.

🗄️

Проектування БД

Реляційна модель (PostgreSQL), нормалізація до 3НФ, індексування для прискорення запитів рейтингових таблиць та сіток турніру.

🌐

Веб-технології

SPA-архітектура на React, серверна частина Node.js/Express, WebSocket для live-оновлень сітки, Docker для контейнеризації.

📊

Методи оцінки

Навантажувальне тестування (k6), опитування організаторів для оцінки UX, метрики: час операцій, кількість помилок, задоволеність.

05

Очікувані результати

Реалізація проєкту забезпечить повноцінну платформу для проведення турнірів будь-якого масштабу з мінімальними витратами на адміністрування та максимальною прозорістю для учасників.

🏆

Повнофункціональна веб-система

Готовий до розгортання веб-додаток із підтримкою турнірів Swiss, Round Robin та Elimination форматів.

⚡

Скорочення адміністративних витрат

Автоматичне жеребкування, підрахунок очок та формування рейтингу скоротить час адміністратора на ≥60%.

📱

Адаптивний інтерфейс

Зручне використання з будь-якого пристрою: учасники стежать за результатами зі смартфона в реальному часі.

📐

Відкрита архітектура

Модульна структура з REST API дозволяє інтегрувати систему з існуючими клубними сайтами та платформами.

📄

Наукова документація

Кваліфікаційна робота з аналізом алгоритмів жеребкування, порівнянням систем та обґрунтуванням архітектурних рішень.

🧪

Верифіковане тестування

Набір автоматичних тестів (unit + integration) та результати навантажувального тестування системи.
